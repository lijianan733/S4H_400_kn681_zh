Authorization Checks in Updates

No authorization checks can be made during updates (that is when processing an update function module triggered by COMMIT WORK).

  • In updates, the statement AUTHORITY-CHECK always sets the value sy-subrc to 0 and does not make any authorization checks.
  • CDS access control cannot take place. If the ABAP SQL statement SELECT is used in an update to access a CDS entity for which access control is not disabled using the value #NOT_ALLOWED for the annotation @AccessControl.authorizationCheck or using the addition WITH PRIVILEGED ACCESS in the FROM clause, the runtime error SYSTEM_UPDATE_TASK_ILL_STMT occurs.

This applies to local updates and to update sessions.
