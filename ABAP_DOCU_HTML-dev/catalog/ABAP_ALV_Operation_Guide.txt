================================================================================
ABAP OOALV 航班信息报表 - 操作说明与开发指南
================================================================================

【项目概述】
本示例为一个完整的ABAP OO ALV报表，涉及航班信息管理，支持增加、删除、修改航班记录。
相关文档：docs/advanced/abapat*.htm, docs/ui/dynp*.htm

================================================================================
【核心数据结构】
================================================================================

结构名称: ty_flight
  - fldate(日期)      : 航班日期，SFLIGHT-FLDATE格式
  - carrid(航空公司)  : 航空公司代码，如'LH'(汉莎)、'UA'(联合)
  - connid(航班号)    : 连接号，如'0400'
  - custname(客户)    : 客户名称，文本型，max 30字符
  - seatsocc(座位)    : 已占座位数，整数型(INT4)，范围 0-400
  - price(价格)       : 机票价格，货币型，格式 XXX.XX
  - currency(货币)    : 货币代码，如'EUR'、'USD'
  - status(状态)      : 记录状态，'A'=活跃，'C'=取消

结构应用场景：
  1. 输入表: g_flight_tab (STANDARD TABLE，按fldate+carrid+connid为主键)
  2. 备份表: g_flight_backup (用于"取消编辑"恢复)

================================================================================
【OOALV 关键概念】
================================================================================

1. 容器创建（Docking Container）
   - 目的: 在报表输出区域创建可交互的容器
   - 位置: dock_at_bottom/top/left/right
   - 比例: ratio = 75 表示容器占屏幕75%高度

2. ALV网格初始化（cl_gui_alv_grid）
   - set_table_for_first_display: 首次显示时绑定数据表、字段目录、布局参数
   - 重要参数:
     * i_save = 'A': 允许用户保存布局变体
     * is_layout-edit = abap_true: 启用单元格编辑模式
     * is_layout-sel_mode = 'C': 列选择模式

3. 字段目录构建
   - fieldname: 对应结构中的字段名
   - coltext: 列标题
   - edit: 是否可编辑（KEY字段通常edit=false）
   - datatype: 数据类型，影响排序和显示格式
   - width: 列宽度(字符单位)

4. 事件处理机制
   - SET HANDLER: 注册事件处理器
   - 常见事件:
     * toolbar: 工具栏按钮点击
     * data_changed: 用户修改单元格
     * user_command: 用户快捷键或菜单命令
     * double_click: 双击行

【操作流程说明】
================================================================================

【新增行 (ADD_ROW)】
  步骤：
    1. 创建新的ty_flight结构实例
    2. 填充默认值（日期+3天、同一航空公司、新航班号'9999'、默认客户名'New Passenger'）
    3. INSERT到内表索引1（插到顶部，便于用户编辑）
    4. 调用refresh_table_display刷新显示
    5. 显示成功消息

  关键点：
    - 新行插入位置选择：INSERT可选择索引1(顶部)或APPEND(末尾)
    - 默认值设置应符合业务逻辑
    - 刷新后ALV自动跳转到新行，方便用户立即编辑

【删除行 (DELETE_ROW)】
  步骤：
    1. 获取用户选中的行: g_alv->get_selected_rows(et_index_rows = lt_rows)
    2. 按INDEX倒序排序: SORT lt_rows BY index DESCENDING
    3. 循环删除: DELETE g_flight_tab INDEX ls_row-index
    4. 刷新显示

  关键点：
    - 倒序删除的原因：如果正序删除，删除第1行后，原第2行变为第1行，后续索引会错乱
    - 例: 删除行1、2、3，正序操作会导致遗漏；倒序则先删3再删2再删1，索引不受影响
    - get_selected_rows返回LVC_T_ROW表，包含INDEX和SELID

【修改数据 (EDIT)】
  步骤：
    1. 用户在ALV单元格中直接输入或修改值
    2. 触发data_changed事件
    3. 在事件处理中进行数据校验
    4. 若校验失败，返回错误消息（阻止保存）
    5. 若校验通过，允许修改保存到内表

  可编辑字段（edit=true）：
    - custname: 客户名称（字符型）
    - seatsocc: 座位数（整数型，需验证范围0-400）
    - price: 价格（货币型，需验证非负）
    - status: 状态（'A'或'C'）

  不可编辑字段（edit=false）：
    - fldate, carrid, connid: KEY字段，通常不允许修改

【保存数据 (SAVE_DATA)】
  步骤：
    1. 循环遍历g_flight_tab中所有记录
    2. 验证必填字段（custname、seatsocc、price不为空）
    3. 若有错误则返回错误消息，阻止保存
    4. 若全部通过，执行实际数据库操作(MODIFY sflight FROM ls_flight)
    5. COMMIT WORK AND WAIT确保数据持久化
    6. 显示"数据保存成功"消息

  实际应用中的考虑：
    - 错误处理：使用TRY-CATCH捕获数据库异常
    - 事务管理：大批量修改时考虑COMMIT WORK AND WAIT的性能影响
    - 日志记录：记录修改前后的值便于审计（MODIFY aude_log）

【取消编辑 (CANCEL_EDIT)】
  步骤：
    1. 将g_flight_tab恢复为g_flight_backup的副本
    2. 刷新显示
    3. 消息提示"已取消修改"

  设计原因：
    - 用户在修改过程中可能改变主意
    - 备份表在START-OF-SELECTION时生成：g_flight_backup = g_flight_tab
    - 不涉及数据库回滚，仅是内表数据恢复

================================================================================
【数据验证逻辑】
================================================================================

【座位数验证 (SEATSOCC)】
  规则: 座位数必须在0-400之间
  触发事件: data_changed (当修改SEATSOCC字段时)
  实现代码:
    IF ls_flight-seatsocc < 0 OR ls_flight-seatsocc > 400.
      MESSAGE 'Seats must be between 0 and 400' TYPE 'E'.
      RETURN.
    ENDIF.
  类型: 'E' = 错误，阻止继续操作

【必填字段验证 (Save前)】
  规则: custname、seatsocc、price都不能为空
  触发时机: 用户点击"保存"按钮
  实现: LOOP AT g_flight_tab + IF初始检查

【状态值验证 (可选)】
  规则: status只能是'A'(活跃)或'C'(取消)
  实现: 可在data_changed或F4帮助中限制输入

【关键ABAP语法点】
================================================================================

1. APPEND与INSERT的区别
   APPEND: 添加到表末尾，不能指定索引
   INSERT: 可指定INDEX，灵活性更强

2. 倒序排序与删除
   SORT lt_rows BY index DESCENDING.
   原因：正序删除会导致索引错乱（删除后元素上移）

3. 字段符号与引用
   LOOP AT g_flight_tab INTO DATA(ls_flight).
   DATA(...)是隐式字段符号创建，推荐使用

4. VALUE 构造器
   APPEND VALUE ty_flight(...) TO g_flight_tab.
   现代ABAP语法，比传统CLEAR + MOVE更简洁

5. 内表初始化
   CLEAR g_flight_tab.    "清空整个内表
   g_flight_backup = g_flight_tab.  "副本赋值

6. 系统字段
   sy-datlo: 当前逻辑日期（类似sy-datum但考虑时区）
   sy-repid: 当前报表ID
   sy-dynnr: 当前屏幕号（报表输出屏为'100'）

================================================================================
【常见错误与解决】
================================================================================

错误1: 删除行后索引错乱
  原因: 正序删除导致后续行索引前移
  解决: 使用倒序排序后再删除

错误2: ALV显示不刷新
  原因: 数据修改后未调用refresh_table_display
  解决: 任何修改表数据的操作后都需调用refresh

错误3: 选中行为空
  原因: get_selected_rows没有检查返回值
  解决: 先检查lt_rows是否有内容，再处理

错误4: F4帮助不可用
  原因: 字段目录中未设置F4帮助相关参数
  解决: 在字段目录中设置rollname或f4checktable

错误5: 编辑模式不生效
  原因: 字段目录中edit=false或layout中edit=false
  解决: 检查两处设置，确保都启用编辑

【性能优化建议】
================================================================================

1. 大数据量处理（>10000行）
   - 使用ROWS限制初始加载量
   - 实现懒加载机制（用户滚动时加载）
   - 考虑使用过滤条件减少行数

2. 内表操作性能
   - 优先使用STANDARD TABLE（KEY查询快）
   - 避免频繁INSERT（改用APPEND后SORT）
   - 使用READ TABLE WITH KEY替代LOOP

3. ALV刷新性能
   - 不要频繁调用refresh_table_display
   - 考虑使用refresh_data_changed_finished减少刷新范围
   - 批量修改时收集所有变化后一次性刷新

4. 内存管理
   - 及时CLEAR不需要的临时变量
   - 考虑使用SORT...AS指令避免额外排序

【开发流程】
================================================================================

第1步: 需求分析
  - 确定数据结构（哪些字段、数据类型）
  - 定义操作（增删改查）
  - 规划验证规则

第2步: 创建TYPES
  - 定义结构: BEGIN OF ... END OF
  - 定义内表: TYPE STANDARD/SORTED/HASHED TABLE

第3步: 实现选择屏
  - SELECTION-SCREEN参数（查询条件）
  - AT SELECTION-SCREEN验证

第4步: 实现数据加载
  - SELECT读取数据库
  - 如无数据则插入示例
  - 创建备份表

第5步: 实现ALV显示
  - 创建容器
  - 创建ALV网格
  - 构建字段目录
  - 设置事件处理

第6步: 实现CRUD操作
  - 新增：INSERT + refresh
  - 删除：get_selected_rows + DELETE + refresh
  - 修改：data_changed事件验证
  - 保存：检查 + MODIFY + COMMIT

第7步: 测试
  - 单一行操作测试
  - 批量操作测试
  - 边界值测试
  - 错误处理测试
