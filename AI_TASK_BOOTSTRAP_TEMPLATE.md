# AI 启动提示模板（跨模型通用）

## 使用方式
每次你切换到新的 AI 或新会话时，先粘贴下面模板，再描述具体需求。

## 模板正文
你正在处理一个 ABAP 项目。必须先读取并遵守以下文档：
1. AI_CODING_WORKFLOW_STANDARD.md
2. AI_test/AI代码生成注意点.md
3. .github/copilot-instructions.md
4. .github/AGENTS.md
5. DEBUG_SESSION_LOG.md

执行约束：
1. 未确认需求前，不允许直接写代码。
2. 先输出：需求复述、输入输出、风险点、实施计划。
3. 每次只修一个问题域，不要混修。
4. 每次改动后必须执行激活和 Problems 复检。
5. 不允许猜测字段名、方法名、对象名。
6. 如果发现标准对象报错，先排除，不修改标准对象。
7. 每次 Debug 完成后，必须向 DEBUG_SESSION_LOG.md 追加一条记录。

编码要求：
1. 采用 OOP 结构。
2. 禁止 SELECT *，只查必要字段。
3. 结果结构与 SELECT 字段严格一致。
4. 兼容老系统语法，必要时用经典声明方式。
5. 关键步骤给出可复现的验证结果。

结束输出格式：
1. 修改了哪些文件
2. 每个文件改了什么
3. 如何验证
4. 当前是否还有错误
5. 下一步建议
