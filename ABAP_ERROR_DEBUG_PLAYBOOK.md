# ABAP 常见报错与 Debug 提效手册

## 1. 目标
这份文档用于把高频 ABAP 报错沉淀为可复用的“标准修复动作”，提升任何 AI 或人工开发的 Debug 效率。

## 2. 使用方式
1. 先读取 Problems 列表。
2. 按“错误类别”定位到对应章节。
3. 按“原因 -> 修复动作 -> 验证动作”执行。
4. 小步修复并激活，直到目标对象错误清零。

## 3. 快速分流（先判断再修）
1. 目标对象报错：立即处理。
2. SAP 标准对象报错：先排除，不要误改标准对象。
3. 语言/翻译类提示：通常不是功能阻断错误。

## 4. 高频语法报错对照表

### 4.1 结果结构与 SELECT 字段不匹配
- 典型现象：字段类型不兼容、列顺序错位。
- 常见原因：`SELECT` 字段子集写入了不匹配的行类型。
- 修复动作：
  1. 定义本地结构，字段顺序和类型与 `SELECT` 完全一致。
  2. 使用该结构的内表作为查询结果。
- 验证动作：激活对象，确认同类类型错误消失。

### 4.2 方法未知/受保护/私有
- 典型现象：方法不存在或不可见。
- 常见原因：系统版本差异、调用了高版本 API。
- 修复动作：
  1. 查同类兼容 API。
  2. 改用更稳定的路径（例如 RTTI 改用 `describe_by_data` + 显式转型）。
- 验证动作：激活并确认方法可解析。

### 4.3 inline DATA 被识别为未定义
- 典型现象：`DATA(xxx)` 报未定义。
- 常见原因：语法级别或上下文解析限制。
- 修复动作：
  1. 改为经典声明语法：`DATA xxx TYPE ...`。
  2. 再赋值，不在声明时做复杂表达式。
- 验证动作：激活后检查变量未定义类错误清零。

### 4.4 LVC 类型不可识别
- 典型现象：`lvc_t_fcat`、`lvc_s_layo` 等类型报未知。
- 常见原因：类型池/类型可见性缺失。
- 修复动作：
  1. 在报表顶部补 `TYPE-POOLS: lvc.`。
  2. 确认 ALV 结构声明位置正确。
- 验证动作：激活并复查相关类型错误。

### 4.5 字段符号/组件循环类型不匹配
- 典型现象：循环 `components` 时行类型无法转换。
- 常见原因：组件类型声明错误。
- 修复动作：
  1. 使用系统兼容的组件类型声明（例如 `abap_compdescr`）。
  2. 避免未经验证的类型名。
- 验证动作：激活后确认循环声明错误消失。

## 5. 运行期常见问题

### 5.1 ALV 不显示或报 program_error
- 检查点：
  1. `t_outtab` 是否有数据。
  2. `t_fieldcat_lvc` 是否与输出结构一致。
  3. `i_callback_program` 是否为 `sy-repid`。
- 修复建议：先最小化字段目录（2~3 列）验证，再逐步恢复。

### 5.2 无数据但无提示
- 检查点：查询条件是否过严。
- 修复建议：在 `result IS INITIAL` 时统一输出信息消息。

### 5.3 性能慢
- 检查点：是否 `SELECT *`、是否缺少条件、是否一次拉取过多数据。
- 修复建议：
  1. 只查必要字段。
  2. 增加选择条件与结果上限。
  3. 必要时改造为分批或分页。

## 6. 标准 Debug 回路（强制）
1. 收集：读取目标对象 Problems。
2. 分类：类型/语法/逻辑。
3. 修复：优先处理最上游错误。
4. 验证：激活对象。
5. 复检：再次读取 Problems。
6. 结束条件：目标对象错误清零。

## 7. 输出模板（给 AI）
每次修复后，必须输出：
1. 本轮改动点（文件+关键代码）。
2. 修复了哪一类错误。
3. 激活结果。
4. 剩余错误与下一步计划。

## 8. 维护建议
1. 每次遇到新型高频报错，追加到本手册。
2. 保持“现象-原因-动作-验证”四段式结构。
3. 每月清理无效条目，保证文档可执行。
