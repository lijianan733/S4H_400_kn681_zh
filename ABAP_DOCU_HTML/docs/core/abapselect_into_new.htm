SELECT - INTO NEW

Quick Reference

Syntax

... INTO NEW @wa
  | (..., NEW @dobj, ...)
  | NEW @itab
  | NEW @DATA(wa)
  | NEW @DATA(itab) ...

Alternatives:

1. ... INTO ... NEW @wa ...

2. ... INTO ... (..., NEW @dobj, ...) ...

3. ... INTO ... NEW @itab ...

4. ... INTO ... NEW @DATA(wa) ...

5. ... INTO ... NEW @DATA(itab) ...

Effect

The addition NEW of the INTO clause is used to create the target area as an anonymous data object to which wa and itab point as data reference variables. The data reference variables can exist already or can be declared inline. In this case, inline declarations are also possible together with dynamic tokens and after FETCH. The addition NEW cannot be specified together with APPENDING.

Alternative 1

... INTO ... NEW @wa


Alternative 2

... INTO ... (..., NEW @dobj, ...)


Alternative 3

... INTO ... NEW @itab ...


Effect

Creates an anonymous data object with existing data reference variables. The data reference variables must be typed appropriately or can be typed generically with REF TO data. The typing of non-generic data reference variables must satisfy the requirements for the target areas of the INTO clause without the addition NEW.

Alternative 4

... INTO ... NEW @DATA(wa)


Alternative 5

... INTO ... NEW @DATA(itab)


Effect

Creates an anonymous data object with inline declaration of the data reference variables. The data reference variables are typed statically with REF TO data. The dynamic type is a structure or internal table that, like in the case of an inline declaration, is determined without the addition NEW.
