SAP LUW

Related sequence of programming units, whose execution , for example in dialog steps or in remote function calls, can be distributed across multiple work processes, but whose database changes are updated within a single database LUW. SAP LUWs are implemented by bundling techniques, in which programming units such as update function modules or subroutines are registered in different work processes but executed by a single work process. An SAP LUW is usually started by opening a new internal session and completed by the ABAP SQL statement COMMIT WORK. Changes within one SAP LUW can be undone by the ABAP SQL statement ROLLBACK WORK.
